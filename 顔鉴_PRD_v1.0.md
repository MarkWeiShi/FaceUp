## 一、产品概述

### 1.1 产品背景

东亚地区（中日韩越）存在高度统一的审美文化体系，「颜值经济」渗透护肤、彩妆、医美、服装等多个赛道，年市场规模超 5000 亿人民币。用户对颜值的关注度和讨论深度显著高于欧美市场。

**现有同类产品的核心痛点：**

| 竞品 | 优点 | 核心不足 |
|------|------|---------|
| FaceApp | 变脸效果丰富 | 评分基于西方模型，无东亚维度 |
| Meitu 美图测颜值 | 国民认知度高 | 报告维度单一，无改善方案 |
| 小红书「测颜值」话题 | UGC 社区热度高 | 非标准化，结果不可复现 |
| 韩国 SNOW 颜值评分 | 韩式美学契合 | 无护肤/医美建议，仅娱乐属性 |

**顔鉴的差异化定位：**

> 专为东亚审美校准的 AI 颜值分析工具 —— 不只给分，更给出路。

产品以 **0-9 分制**评分、**8 维度东亚美学体系**、**护肤 × 彩妆 × 医美三层改善方案**为核心差异化，打通「评估 → 诊断 → 改善」完整链路。

### 1.2 产品目标

| 维度 | 目标 |
|------|------|
| 用户目标 | 获得基于东亚美学的专业颜值分析，并得到可执行的个性化改善方案 |
| 产品目标 | 打造东亚颜值垂直赛道标杆 H5 工具，3 个月内 DAU 破 10 万 |
| 商业目标 | 通过报告内容引导护肤/医美消费决策，形成 B2B2C 变现闭环 |
| 技术目标 | 首屏加载 < 2s，人脸分析响应 < 4s，H5 全平台适配 |

### 1.3 评分机制说明（0-9 分制）

**为什么用 0-9 分？**

- 与东亚用户日常语言高度匹配（「她是个9分女神」「7分算不错了」已是文化梗）
- 区分度清晰，避免 100 分制集中在 60-80 区间造成的数值钝化
- 小数位（如 7.2）提供精细感，增强报告的科学可信度
- 参考：小红书颜值讨论中 91% 的用户自发使用 1-10 分制（数据来源：2024 小红书内容分析）

**段位体系：**

| 分数区间 | 段位标签 | 东亚文化映射 | 预计用户比例 | 颜色标识 |
|---------|---------|------------|------------|---------|
| 8.5 – 9.0 | ✦ 神颜 | 明星级、登顶颜值金字塔 | ~1% | `#F0C060` 金 |
| 7.5 – 8.4 | ◈ 高颜值 | 公认好看，有明显五官优势 | ~9% | `#C9A96E` 香槟金 |
| 6.5 – 7.4 | ◇ 靓丽 | 颜值在线，特征鲜明 | ~20% | `#7EB8A4` 玉绿 |
| 5.5 – 6.4 | ○ 清秀 | 整体协调，有一定亮点 | ~30% | `#A8C5D8` 天青 |
| 4.5 – 5.4 | △ 邻家 | 平和耐看，有提升空间 | ~25% | `#D4849A` 玫瑰 |
| 4.0 – 4.4 | ◎ 潜力股 | 可塑性强，改善空间大 | ~15% | `#B0A8B0` 银灰 |

> ⚠️ **产品策略：** 实际输出分数下限为 **4.0**，低于此值统一显示「潜力股」标签，避免负面情绪影响用户留存。

#### 3.3.2 图片质量预检逻辑

上传/拍摄后在进入分析前执行前端预检：

| 检测项目 | 检测方法 | 阈值 | 不通过时提示文案 |
|---------|---------|-----|---------------|
| 人脸数量 | Face++ 检测 API | 需有且仅有 1 张人脸 | 「未检测到人脸，请重新上传正面照」 |
| 正脸角度 | Yaw 偏转角 | ｜yaw｜< 25° | 「请正面面对摄像头，侧脸影响准确度」 |
| 图片清晰度 | 拉普拉斯方差 | > 80 | 「图片有些模糊，建议重新拍摄」 |
| 图片亮度 | 平均像素亮度 | 40–220（0-255） | 「光线不足/过曝，建议补光后重拍」 |
| 遮挡检测 | 关键点可见度 | 眼/鼻/嘴可见度 > 0.6 | 「面部被遮挡，请摘下口罩/墨镜」 |
| 图片尺寸 | 文件大小 | > 50KB，分辨率 > 480px | 「图片分辨率过低，建议换一张」 |

**预检通过率优化策略：**
- 弱光检测：当亮度 < 60 时，仅提示建议（黄灯），不强制阻断
- 允许轻微侧脸：yaw 25°–35° 之间给出建议而非硬拒
- 预检 UI：在图片预览上叠加扫描线动画（0.8s），预检失败时扫描线变红

### 4.2 核心评分维度（东亚美学体系）

#### 4.2.1 8 维度评分矩阵

| 编号 | 评分维度 | 英文 Key | 具体检测指标 | 东亚权重 | 西方权重 | 评分算法 |
|-----|---------|---------|------------|---------|---------|---------|
| D1 | 面部对称性 | symmetry | 左右半脸 68 关键点欧氏距离偏差系数 | 12% | 18% | 偏差越小分越高，阈值 <3px 得满分 |
| D2 | 三庭五眼比例 | proportion | 上中下庭高度比（理想 1:1:1），五眼宽度分布 | 15% | 10% | 接近黄金比例 0.618 的程度 |
| D3 | 脸型轮廓 | faceShape | V脸程度（下颌角）、颧骨突出度、脸宽/脸长比 | 14% | 8% | 下颌角 < 110° 加分，脸宽/脸长 < 0.75 加分 |

#### 4.2.3 评分计算公式

```
维度分 Di = 原始检测值 → 标准化至 [0, 9] 区间（各维度有独立 sigmoid 函数）
加权总分 = Σ(Di × Wi)  其中 Wi 为各维度权重
最终分 = max(加权总分, 4.0)  // 底线保护
```

## 十一、技术方案

### 11.1 技术架构

| 层级 | 技术选型 | 选型理由 |
|-----|---------|---------|
| 前端框架 | 原生 HTML + CSS + Vanilla JS | H5 场景无需框架，减少包体，First Load < 150KB |

| AI 文案生成 | Anthropic Claude claude-sonnet-4-6 | 多语言（中日韩越）高质量文本，Vision 能力强 |
| 图表渲染 | Canvas 2D API（原生） | 避免引入 Chart.js 等库，雷达图自实现 |
| 图片压缩 | browser-image-compression (npm) | 前端压缩，目标 < 600KB |
| 本地存储 | IndexedDB（历史）+ localStorage（设置） | 无需登录，端侧数据 |
| 分享图片 | html2canvas | 生成分享卡片 |
| 字体加载 | Google Fonts (Noto Serif SC + Cormorant Garamond) | CDN 加速，font-display: swap |

### 11.2 API 调用流程

```

    ↓

> 📌 **核心承诺（首次使用时全文展示，需用户点击同意）：**

- **传输加密：** 所有 API 请求使用 HTTPS/TLS 1.3
